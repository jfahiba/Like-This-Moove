Leaphly cart Shopping cart built around RESTful specs. Shopping cart thought for developers. Shopping cart as a service.

Mission Statement

The Leaphly project makes it easier for developers to add cart functionality to php application made with Symfony2 or to those application that could consume REST API.

This software provides the tool and some guideline for building decoupled, high-quality and long-life e-commerce application.

Features

Multi-product

Support infinite products each with its personal domain.

Multi-storage

Doctrine2 ODM and ORM.

No EAV

Entity Attr. Value
Object Oriented Programming

RESTful API.

Usable via Http RESTful or via Symfony2 container.

No Session needed.

Cart is used as a resource.

States and Transitions are not constants, are defined by you.

Your domain is the boss here.

A PHP shopping cart standing on the shoulder of giants

Leaphly is a PHP application for PHP 5.3. It is built on the shoulders of Symfony2 and is also inspired by Spree commerce, symfony-cmf and by the symfony ecosystem.

A great e-commerce solution should be:

Tailored should not be configured only with an ".ini" file, it needs to be developed, in order to fit the company needs.
Deep should handle each families of product with its own handler.
Decoupled should facilitate and suggest the SOA, with internal or external services: 'Cart', 'User', 'Product', 'Purchase process', 'Reporting' (...).
Flexible should let the architectural choice to the development team, CRM instead of user bundle, CMF for product managment, Business Intelligence tool for reporting...
